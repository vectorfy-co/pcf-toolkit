# Accessibility

This documentation site aims to meet WCAG 2.1 AA.

## Features

- High-contrast color palette with dark/light themes
- Visible focus styles for keyboard navigation
- Semantic HTML structure from Markdown
- Responsive layout for mobile and desktop
- Print-friendly styling

## Tips for authors

- Use clear heading levels (H2 -> H3 -> H4)
- Provide alt text for images
- Avoid color-only meaning in tables or diagrams

If you spot an accessibility issue, please report it via GitHub issues.
